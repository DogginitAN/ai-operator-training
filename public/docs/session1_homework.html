PREREQ Install Git for Windows (required for Claude Code)

Claude Code needs Git installed to work on Windows. You started this on the call — here's the full walkthrough to make sure it's done.

STEP 1 — DOWNLOAD
Go to git-scm.com/download/win and download the installer. It should auto-detect your system (64-bit).
STEP 2 — INSTALL
Run the installer. Click Next on every screen — the defaults are all fine. The important thing is that "Add Git to PATH" stays checked (it's checked by default, don't uncheck it).
STEP 3 — VERIFY
Close your PowerShell / terminal completely, then open a fresh one. Type:
git --version
You should see something like git version 2.47.1.windows.1. If you see that, Git is installed.
STEP 4 — VERIFY CLAUDE CODE
In that same fresh terminal, type:
claude --version
If this works, you're good. If it says "not recognized," run the installer again:
irm https://claude.ai/install.ps1 | iex
Then close and reopen your terminal and try claude --version again.
The #1 Windows gotcha: After installing anything, you MUST close and reopen your terminal for it to take effect. This is a Windows PATH thing — it's not you, it's Windows.
ASSIGNMENT Build a Tip Calculator App

You're going to build a working web app — a tip calculator — using Claude. No coding knowledge required. Claude writes all the code. You just tell it what you want.

There are two ways to do this. Try both so you get a feel for the difference:

Path A — Claude Code window in Claude Desktop

This is the code tab you saw during our call (top of Claude Desktop, next to Chat and Cowork). It's the easiest entry point — no terminal needed.

Path B — Claude Code in your terminal (PowerShell)

This is the terminal version we started setting up. More powerful, more flexible, and where you'll eventually live. But Path A is totally fine to start.

PATH A Build It in the Claude Code Window (Claude Desktop)
STEP 1
Open Claude Desktop. At the top, click the Code tab (next to Chat and Cowork).
STEP 2 — TYPE THIS PROMPT
Build me a simple single-page web app — a tip calculator. I want to enter a bill amount and select a tip percentage (15%, 18%, 20%, 25%, or custom). It should show the tip amount and total. Clean design, works on mobile. Use HTML, CSS, and JavaScript in a single file. Save it to my Desktop and open it in my browser when you're done.
WHAT SHOULD HAPPEN
Claude will create an HTML file, write all the code, save it to your Desktop, and open it in your browser. You'll have a working tip calculator. The whole thing takes about 60 seconds.
THEN TRY THIS
Once it's built, ask Claude to modify it:
Add a "split the bill" feature — let me enter how many people are splitting and show the per-person amount.
This shows you the iteration loop — you describe what you want changed, Claude modifies the code, and you see the result immediately.
PATH B Build It in Your Terminal (PowerShell)

This is the same exercise but from the terminal. Once you're comfortable here, you have the full power of Claude Code at your fingertips.

STEP 1 — OPEN POWERSHELL
Press Win + X, then click Terminal (or Windows PowerShell). You should see a command prompt.
STEP 2 — CREATE A PROJECT FOLDER
Type these two commands, pressing Enter after each:
mkdir $HOME\ai-demo cd $HOME\ai-demo
This creates a folder called ai-demo in your user directory and moves into it.
STEP 3 — START CLAUDE CODE
claude
First time: it'll open your browser to authenticate. Log in with your Claude account (same one as the Max subscription). Once authenticated, you'll see Claude Code's interactive prompt in your terminal.
STEP 4 — TYPE THIS PROMPT
Inside Claude Code, type:
Build me a simple single-page web app — a tip calculator. I want to enter a bill amount and select a tip percentage (15%, 18%, 20%, 25%, or custom). It should show the tip amount and total. Clean design, works on mobile. Save it as index.html in this directory and open it in my browser.
WHAT SHOULD HAPPEN
Claude Code will create the file, write the code, and open it in your browser. Same result as Path A, but you did it from the terminal.
THEN TRY
Add a "split the bill" feature — let me enter how many people and show per-person amount. Also add a dark mode toggle.
USEFUL COMMANDS INSIDE CLAUDE CODE
  • /help — shows all available commands
  • /compact — compresses conversation history (use if it starts feeling slow)
  • Esc — cancel what it's currently doing
  • Type exit to leave Claude Code and go back to normal terminal
HELP If Something Goes Wrong

"claude is not recognized" — Close your terminal completely, reopen it, try again. If still broken, run the installer again:

irm https://claude.ai/install.ps1 | iex

Then close and reopen terminal.

"Git Bash not found" — Make sure Git is installed (Step 1 above). If it is installed but Claude Code still can't find it, you can paste this into Claude Chat and ask for help — screenshot the error message and drop it in.

Claude Code asks to allow permissions — Say yes / always allow. Same as we did on the call with Desktop Commander. You can tighten permissions later.

The app opens but looks wrong or doesn't work — Just tell Claude: "The calculator isn't working correctly — [describe what's wrong]." It'll fix its own code. This is normal. Iteration is the workflow.

Anything else breaks — Screenshot the error, paste it into Claude Chat (not Code), and say "I'm trying to use Claude Code to build an app and hit this error. What should I do?" Claude is excellent at troubleshooting itself.

We have a team of advocates who manage the process. WHY THIS MATTERS
Next time you want to prototype something — a quote comparison tool, a provider scorecard, an intake form — you start a conversation inside this project and Claude already knows the context. This is the Memory layer we talked about. Brain, Hands, Memory.
DAILY This Week's Reps (Before Thursday)

The app is the main homework. But try to work these into your week too:

  • 3 real tasks per day in Claude. Provider emails, customer situations, document review — actual work, not toy examples. Use Chat or Cowork, whichever feels natural.
  • Try Caps Lock voice input at least once. Hold Caps Lock, talk your problem, let go. Notice how different it feels to speak your request vs. type it.
  • Upload something dense. A vendor contract, a long email chain, a competitor's pricing page. Ask Claude to extract the 3 things that matter and flag anything unusual.
  • Use the calendar connector. Ask Claude to look at your week and tell you where you have gaps, or prep you for tomorrow's meetings.
Remember: You're not learning a tool. You're building a reflex. Every time you catch yourself doing something that requires thinking — stop and try handing it to Claude first. Even if it takes longer the first time. The reps matter more than the results right now.
