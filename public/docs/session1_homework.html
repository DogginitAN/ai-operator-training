Session 1 Homework: Your First Build

A guided journey from idea to working app — due before Thursday April 3rd
WHY What This Homework Is Really About

This is not a checklist of disconnected tasks. It's a single guided journey that walks you through the entire AI-native workflow — from creating a project, to planning with Claude, to building a working app, to iterating on it. Each step unlocks the next.

By the end, you'll have experienced every layer of the system we talked about on the call:

  • Brain — Claude thinks with you in Chat (planning the app)
  • Hands — Claude Code builds for you in the terminal (writing the code)
  • Memory — Your Project carries context forward (persistent knowledge)

That's the whole model. This homework makes it real.

Total time: About 45 minutes. You can do it in one sitting or break it into pieces.
STEP 1 Install Git for Windows

Why this matters: Claude Code uses Git under the hood to track changes to your code. Without Git, Claude Code can't run on Windows. This is the one prerequisite — everything else depends on it.

DOWNLOAD & INSTALL
Go to git-scm.com/download/win and download the installer. Run it — click Next on every screen (defaults are fine). Make sure "Add Git to PATH" stays checked.
VERIFY
Close PowerShell completely, open a fresh one, and type:
git --version
Then verify Claude Code:
claude --version
If claude isn't recognized, reinstall: irm https://claude.ai/install.ps1 | iex — then close and reopen terminal.
Windows gotcha: After installing anything, you MUST close and reopen your terminal. This is how Windows updates the PATH. It's not you — it's Windows.
You just set up the foundation that lets an AI write, run, and manage code on your machine. This is what makes Claude Code different from ChatGPT — it doesn't just suggest code, it executes it.
STEP 2 Create the "Building with AI" Project

Why this matters: On the call, we created the AI Operator Training project to track our sessions. Now you're creating a project for building things. This is the Memory layer — every conversation inside this project carries your context forward. Without a project, every new chat starts from zero.

CREATE IT
Claude Desktop → Sidebar → Projects → Create New Project → Start from scratch.

Name it: Building with AI
ADD THESE INSTRUCTIONS
This project is for building tools, apps, and prototypes. About me: I'm Matt Doumar, founder of HouseAccount — a home services marketplace that connects homeowners with service providers (plumbers, electricians, etc). We have a team of advocates who manage the process. When building in this project: - Keep it simple and functional first, then iterate - Use HTML/CSS/JS for web tools (single file when possible) - Explain what you're doing in plain language so I can learn - I'm on Windows using PowerShell - I'm learning — walk me through decisions, not just code
You just gave Claude a permanent onboarding packet. Every future conversation in this project — tomorrow or next month — Claude already knows your context. A model with no context is smart but useless. A model with your context is a teammate.
STEP 3 Plan the App in Claude Chat

Why this matters: Before we build anything, we think. This is the Brain layer. You're going to use Claude Chat (inside your new project) to plan what you want to build. Real AI-native operators don't just start coding — they have a thinking partner help them scope it first.

OPEN A CHAT INSIDE YOUR PROJECT & TYPE THIS
I want to build my first web app as a learning exercise. It should be a tip calculator — enter a bill amount, pick a tip percentage (15%, 18%, 20%, 25%, or custom), and see the tip amount and total. I also want a "split the bill" feature where I enter how many people and it shows per-person amounts. Before we build it, help me think through: 1. What should the UI look like? Describe the layout. 2. What inputs and outputs do I need? 3. Any edge cases to handle (like empty inputs, zero bill)? Then give me the exact prompt I should paste into Claude Code to build it.
WHAT HAPPENS
Claude will think through the design with you and hand you a ready-to-go prompt. Save that prompt — you'll use it in Step 5.
BONUS MOVE
While in this chat, ask Claude to suggest a simple HouseAccount-related app you could build next — a quote comparison tool, a provider rating card, an intake form. You don't have to build it yet. Just plant the seed.
You just used Claude as a thinking partner, not a code monkey. Chat is for thinking. Code is for doing. The plan is the bridge between them.
STEP 4 Set Up the Project Folder (Terminal)

Why this matters: Claude Code works inside a directory on your machine. You need to create one. Two commands — you'll use them hundreds of times.

OPEN POWERSHELL AND TYPE
mkdir $HOME\tip-calculator cd $HOME\tip-calculator
That's it. You created a folder and moved into it. This is where Claude Code will build your app.
The terminal isn't scary. It's just a text-based way to tell your computer what to do. Two commands — make a folder, move into it.
STEP 5 Build the App in Claude Code (Terminal)

Why this matters: This is the Hands layer. You're about to watch Claude write a complete working application from a single description. No coding knowledge required. This is the moment where "I wish I had a tool that does X" stops being a fantasy and starts being a conversation.

START CLAUDE CODE
claude
First time: your browser opens to authenticate. Log in with your Max account.
PASTE THE PROMPT FROM STEP 3
Use the prompt Claude Chat gave you. If you skipped Step 3, use this:
Build me a simple single-page web app — a tip calculator. I want to enter a bill amount and select a tip percentage (15%, 18%, 20%, 25%, or custom). It should show the tip amount and total. Add a "split the bill" feature. Clean design, works on mobile. Save it as index.html in this directory and open it in my browser.
WHAT HAPPENS
Claude Code creates the file, writes all the code, saves it, and opens your browser. Working tip calculator in about 60 seconds. From one paragraph of English.
You just built software by describing what you wanted. No programming. No developer. No 6-week sprint. A sentence became an app. Now imagine doing this with a quote comparison tool, an intake form, or a provider scorecard for HouseAccount.
STEP 6 Iterate in the Code Tab (Claude Desktop)

Why this matters: Building is never one-shot. The real workflow is: build → test → describe what to change → Claude modifies → test again. This iteration loop is how real software gets made. Now you'll experience it using the Code tab in Claude Desktop — the visual version of what you just did in the terminal.

OPEN CLAUDE DESKTOP → CODE TAB
Ask it to improve your app:
Open the tip calculator I just built at $HOME\tip-calculator\index.html. Add a dark mode toggle and make the design more polished — add smooth animations, better colors, and a modern feel.
TRY ONE MORE
Now add a receipt feature — after calculating, show a summary I could screenshot and send: "Bill: $84.50 / Tip: $16.90 / Total: $101.40 / Per person: $33.80"
This is the feedback loop. Describe → Build → Refine. You never need to get it right on the first try. The terminal and the Code tab are two interfaces to the same power — use whichever feels natural.
STEP 7 Daily Reps — Build the Reflex

Why this matters: The app was the exercise. The real homework is changing your default behavior. Every time you catch yourself doing something that requires thinking — drafting an email, analyzing a document, comparing options — that's a signal to test Claude with it.

  • 3 real tasks per day in Claude. Provider emails, customer situations, document review — actual HouseAccount work. Use Chat or Cowork.
  • Try Caps Lock voice input at least once. Hold Caps Lock, talk your problem, let go.
  • Upload something dense. A vendor contract, a long email chain. Ask Claude to extract the 3 things that matter.
  • Use the calendar connector. Ask Claude to prep you for tomorrow's meetings.
The goal isn't to memorize prompts. It's that by Thursday, your brain starts pattern-matching on "this is a Claude moment" — the same way you instinctively reach for Google when you need to search. That reflex is worth more than any single thing you build.
HELP If Something Goes Wrong

"claude is not recognized" — Close terminal, reopen, try again. Still broken: irm https://claude.ai/install.ps1 | iex then reopen.

"Git Bash not found" — Git isn't installed. Go back to Step 1. Screenshot the error and paste it into Claude Chat.

Claude Code asks permissions — Say yes / always allow. Tighten later.

App looks wrong — Tell Claude: "This isn't working — [describe the problem]." It fixes its own code. Iteration is the workflow.

Anything else — Screenshot it, paste into Claude Chat, say "I hit this error. What should I do?"

The meta-skill: When something breaks, your first instinct should be to ask Claude to fix it. That IS the reflex we're building.
