Claude Code Quick Reference

Every command and shortcut you need — nothing you don't
LAUNCH Starting & Resuming Sessions

How you launch Claude Code determines what it knows, what it can do, and how it behaves. Always cd into your project folder first.

claudeStart a new interactive session in the current folder
claude "your prompt here"Start a session with an opening prompt already loaded
claude -cContinue your most recent session in this directory
claude --resumePick from a list of past sessions to resume
claude -p "query"One-shot mode — run a prompt, print the answer, exit (no interactive session)
claude --versionCheck which version you're running
claude updateUpdate Claude Code to the latest version
Windows rule: After installing or updating anything, close and reopen your terminal. Windows doesn't reload PATH in an existing window.
FLAGS Launch Flags — Set Behavior Before the Session Starts

Flags go after claude when you start a session. They configure how the session runs. You can combine multiple flags.

--dangerously-skip-permissionsAuto-approve all file writes and commands. Use on your personal machine for trusted tasks. The name is a reminder — you're giving Claude full autonomy
--model sonnetStart with a specific model. Use sonnet or opus as shorthand, or the full model name
--effort highSet thinking effort: low, medium, high, or max (Opus only). Higher = slower but more thorough
--add-dir ../other-folderGive Claude access to additional directories beyond the current one
--permission-mode planStart in plan mode — Claude thinks and plans but won't execute until you approve. Options: default, plan, acceptEdits, bypassPermissions
--worktree feature-nameStart Claude in an isolated git worktree — great for parallel work without branch conflicts

Common combos:

claude --dangerously-skip-permissions --model opusFull autonomy with the smartest model — use for complex builds on your machine
claude --permission-mode plan --model opusThink-before-acting mode with Opus — for architecture and planning work
CONTEXT Context Window Management — Your #1 Skill

The context window is Claude's working memory. Everything you say, everything Claude says, every file it reads — all takes up space. When it fills up, quality degrades. These commands help you manage it.

/statusYour fuel gauge — shows context window usage, model, version, and account info. Check this regularly
/contextVisual grid showing exactly what's eating your context. Gives optimization suggestions
/compactCompress the conversation — keeps meaning, frees space. Use at 50% context. You can add focus instructions: /compact focus on the auth module
/clearWipe conversation history completely. Files on disk stay intact. Use when switching tasks in the same project
/costShow token usage and cost for the current session
Rules of thumb: Check /status periodically. Compact at 50%. Start thinking about a fresh session at 75%. Never paste a whole document when you can paste the relevant section.
MODEL Model & Effort — Mid-Session Switching

You can change models and effort levels without restarting your session.

/modelOpen interactive model picker. Use arrow keys to select. Takes effect immediately
/model opusSwitch directly to Opus (smartest, slower). Also accepts full model names
/model sonnetSwitch to Sonnet (fast, capable, good for most tasks)
/effort highSet thinking effort: low (quick answers), medium (default), high (thorough), max (Opus only, deepest reasoning)
/fastToggle fast mode on/off — prioritizes speed over thoroughness
SESSION Session Navigation & History
/resumeBrowse and resume past sessions. Same as launching with claude --resume
/diffInteractive diff viewer — see what changed in your files, per-turn and overall
/rewindRewind conversation and/or code to a previous point. Your undo button
/copyCopy Claude's last response to clipboard. Add a number for older: /copy 2 gets the second-to-last
/exportExport the full conversation as plain text — useful for handoffs or records
/rename my-projectName the current session so you can find it later with /resume
/exitLeave Claude Code and return to your normal terminal
MEMORY Project Memory — CLAUDE.md & Init

CLAUDE.md is how your project remembers context between sessions. Claude reads it automatically every time you start a session in that folder.

/initScan the current project and generate a starter CLAUDE.md file. Great for getting the memory layer going quickly
/memoryEdit CLAUDE.md files, toggle auto-memory on/off, and view auto-generated memory entries
/add-dir ../other-folderGive Claude access to another directory mid-session (files only, not config)
Memory hierarchy: Global (~/.claude/CLAUDE.md) applies everywhere. Project (./CLAUDE.md) applies to that folder. Both are loaded automatically — global first, then project-specific on top.
KEYS Keyboard Shortcuts — Inside a Session

These work while you're inside an active Claude Code session.

Ctrl+CStop what Claude is currently doing. Conversation stays alive — you can give a new instruction
Ctrl+DExit Claude Code entirely (same as /exit)
Shift+TabCycle through permission modes: default → plan → acceptEdits → (bypassPermissions if enabled). Press twice quickly to jump
Alt+T / Option+TToggle extended thinking on/off
Alt+P / Option+PQuick model picker
Ctrl+GOpen your external editor to compose a longer prompt
TabAutocomplete slash commands (type / then Tab)
↑ / ↓Scroll through your command history
PERMISSIONS Permission Modes — How Much Autonomy Claude Gets

Permission modes control whether Claude asks before acting. You set them at launch or cycle through them with Shift+Tab.

defaultClaude asks before file writes and shell commands. The safe starting point
planClaude thinks and plans but won't execute anything until you approve. Good for architecture
acceptEditsAuto-approves file edits but still asks before running shell commands
bypassPermissionsAuto-approve everything. This is what --dangerously-skip-permissions enables. Personal machine only
/permissionsManage fine-grained allow/deny rules for specific tools mid-session
