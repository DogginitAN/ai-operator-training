CONTEXT SKILLS Practice Context Window Management

These skills feel tedious to practice but they prevent the #1 problem every AI operator hits: context degradation. Build the habit now before it costs you.

STEP 1 — SESSION HANDOFFS — DO AT LEAST 2 THIS WEEK
At the end of any productive Claude Chat conversation, before starting a new topic or when the chat is getting long, ask Claude:
Summarize this entire conversation — what we discussed, what was decided, what was built, and what's still open. Format it as a session handoff I can paste into a fresh chat.
Copy the output. Start a new chat (inside the same Project if relevant). Paste the handoff. Continue working. Fresh context, zero lost knowledge.
STEP 2 — /compact — USE IT AT LEAST 3 TIMES
Every time you're in Claude Code this week, make a habit of checking /status. When you cross 50%, run /compact. Do this at least 3 times across your sessions. The goal is to make it automatic — like checking your mirrors when driving.
STEP 3 — RULES OF THUMB
  • Claude Code: /status is your fuel gauge. Compact at 50%. New session at 75%.
  • Chat: new conversation every 30–40 exchanges, or when it feels sluggish.
  • Projects: new chat inside the same Project = fresh context + same persistent memory.
  • Never paste a whole document when you can paste the relevant section.
  • claude --resume in terminal picks up your last Code session.
MEMORY Build Your Project's Memory
STEP 1 — WRITE A CLAUDE.md FOR HOUSEACCOUNT BUILDER
Open your houseaccount-tool folder in the terminal and run:
claude --dangerously-skip-permissions
Then:
/init
This generates a CLAUDE.md file. Edit it — either directly or by telling Claude Code what to add. Include:
  • What HouseAccount does (2–3 sentences)
  • Who the customers are (homeowners) and providers (plumbers, electricians, etc)
  • What tools you've built so far and what they do
  • Your preferences: Windows, PowerShell, single-file HTML apps, keep it simple
  • Any HouseAccount terminology (advocate, job types, etc)
STEP 2 — WHY THIS MATTERS
Every time you start Claude Code in this folder, it reads CLAUDE.md before you say anything. It's the project's memory — onboarding that happens automatically. The more you put in, the less you have to explain each session. This is the "Memory" in Brain, Hands, Memory.

Go to your HouseAccount Builder project in Claude Desktop. Update the project instructions with anything new from this week — pricing ranges you've learned, provider quality criteria, customer segments, common job types. Feed the memory. It compounds.

HELP If Something Goes Wrong

"Claude Code feels slow or confused" — Run /status. If you're past 60%, run /compact. If past 80%, start a fresh session: exit, then claude --dangerously-skip-permissions in the same folder. Claude reads the files on disk, so your project is still there.

"Claude Code keeps asking for permission" — You probably launched without the flag. Exit and relaunch with: claude --dangerously-skip-permissions

"The app Claude built doesn't work or looks wrong" — Don't restart. Just tell Claude what's wrong: "The calculator shows NaN when I enter a number" or "The layout is broken on mobile." It fixes its own code. Iteration is the workflow.

"Context handoff didn't work / new chat doesn't have the context" — Make sure you pasted the handoff text into the new chat. If using a Project, start the new chat INSIDE the project (not outside it). Project instructions carry over; conversation history doesn't.

"/compact didn't free much space" — Compact works best when there's a lot of back-and-forth conversation to compress. If most of your context is file contents, compact won't help as much. In that case, start a fresh session.

"Anything else breaks" — Screenshot the error, paste into Claude Chat (not Code), say: "I'm trying to [what you were doing] and hit this error. What should I do?" Claude is excellent at troubleshooting itself.
