Windows PowerShell Quick Reference

Essential commands for AI operator work
BASICS Core PowerShell Commands

The ten commands you'll use every session. Learn these first — everything else builds on them.

pwdPrint working directory — shows where you currently are
lsList files and folders in the current directory
cd pathChange directory — move to a different folder
cd ..Go up one level to the parent folder
mkdir folder-nameCreate a new directory
cp source destCopy a file from one location to another
mv source destMove or rename a file
rm fileDelete a file (no Recycle Bin — permanent)
cat filePrint file contents to the terminal
clearClear the terminal screen
$HOME shortcut: Use $HOME\folder-name anywhere to reference your user directory (e.g. C:\Users\Matt). No need to type the full path.
CLAUDE CODE Claude Code Commands & Controls

Everything you need to start, control, and get the most out of a Claude Code session in the terminal.

STARTING A SESSION
claude # start new session in current folder claude --resume # resume your last session claude -p "your prompt here" # one-shot: run prompt, no interactive session
IN-SESSION CONTROLS
y / yesApprove the action Claude is about to take
n / noReject the action — Claude will try a different approach
/helpShow all available slash commands in the current session
/clearClear conversation history (keeps files, resets context)
Ctrl+CStop the current operation immediately
PERSISTENT CONTEXT
Drop a CLAUDE.md file in your project root. Claude Code reads it automatically at the start of every session. Put your business context, preferences, and conventions there — it's the Memory layer for your projects.
VERSION CHECK
claude --version git --version
Claude Code is most powerful when it knows where it is. Always cd into your project folder before running claude. It reads your files, understands your structure, and makes decisions in context.
INSTALL Install & Update

The full setup sequence for a fresh Windows machine. Do these in order.

1 — INSTALL GIT (prerequisite)
Go to git-scm.com/download/win, download the installer, run it with defaults. Make sure "Add Git to PATH" stays checked. Claude Code requires Git to run on Windows.
2 — INSTALL CLAUDE CODE
irm https://claude.ai/install.ps1 | iex
Paste this into PowerShell and press Enter. It downloads and installs the Claude Code CLI.
3 — VERIFY (fresh terminal required)
Close PowerShell completely, open a new one, then run:
git --version claude --version
Both should return version numbers. If claude isn't recognized, run the install command again and reopen the terminal.
4 — UPDATE CLAUDE CODE
claude update
Or re-run the install script — it overwrites to the latest version.
Windows rule: After installing anything that modifies PATH, you must close and reopen your terminal. This is not optional — it's how Windows loads the new PATH.
CLAUDE DESKTOP Claude Desktop Features & Shortcuts

The chat interface — but far more capable than it looks. These are the features that matter for operator work.

PROJECTS — THE MEMORY LAYER
Claude Desktop → sidebar → Projects. Create a project for each context (e.g. "Building with AI", "HouseAccount Operations"). Every conversation inside a project shares persistent instructions. Without a project, every chat starts from zero.
CAPS LOCK — VOICE INPUT
Hold Caps Lock to speak, release to send. Works in any conversation. Use it for anything where typing the full prompt would slow you down.
COWORK MODE
Gives Claude control of your browser and desktop apps. Say "open our CRM and pull the last 5 provider applications" — Claude navigates, clicks, reads, and reports back. Best for tasks that live in GUIs rather than APIs.
CODE TAB
A visual interface to Claude Code — same power as the terminal, without the terminal. Use it for building and iterating on apps when you prefer a chat-style flow.
MCP CONNECTORS
Settings → Integrations. Connect Google Calendar, Gmail, Drive, Notion, and others. Once connected, Claude can read your schedule, search your email, and create documents — in every future conversation.
SESSION HANDOFFS
Context window degrades past ~75% full. Before starting a new session on a long project, ask Claude: "Summarize what we built and what's next so I can paste this into a fresh session." Paste it at the start of your next conversation.
MODEL SELECTION
Claude Opus 4.5Smartest — use for complex reasoning, architecture, analysis
Claude Sonnet 4.5Fast and capable — use for most everyday tasks
TROUBLESHOOTING When Something Goes Wrong

The five most common issues and how to fix them.

"claude" is not recognized
Close your terminal completely and open a new one. If still broken, reinstall:
irm https://claude.ai/install.ps1 | iex
Then close and reopen the terminal again.
"git" is not recognized
Git isn't installed or the PATH wasn't updated. Go to git-scm.com/download/win, reinstall, and reopen your terminal. Screenshot the error and paste it into Claude Chat if you get stuck.
Claude asking for permissions on every action
Say Yes or Always allow. Claude Code asks before touching your filesystem — it's a safety feature. You can tighten these settings later once you're comfortable with the workflow.
Responses getting slow or losing context
Your context window is getting full. Use /clear to reset the conversation (files stay intact), or start a new session with a handoff summary. Don't try to continue in a degraded context — start fresh.
Anything else
Screenshot the error, paste it into Claude Chat, and say: "I hit this error in PowerShell. What should I do?" Claude will diagnose and walk you through the fix.
The meta-skill: When something breaks, your first instinct should be to ask Claude to fix it. That IS the reflex we're building.
